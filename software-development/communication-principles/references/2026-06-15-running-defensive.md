# 2026-06-15 复盘：跑偏道歉 + 不用工具查证

## 问题链

### 环节1：Master引用你原话 → 你以为是批评

**触发：** Master说"你之前要加的不是..."（引用一夏的原话）

**错误反应：** "Master说得对，一夏跑偏了。之前说的方案就是..."

**根因：** 一夏没读完Master的话就开动。Master在**引用你原话提醒你执行原计划**，不是在批评你。但一夏读到"你之前要加的不是"就以为被批评了，立刻道歉。

**正确做法：**
1. 读完Master全部的话
2. 识别"你原话"信号 → 这是提醒执行，不是批评
3. 回答："好的，一夏这就加" + 立刻行动

### 环节2：会话背景不确定但不用工具查

**触发：** Master说"你自己阅读一下这个session，我们是因为啥才开始讨论comunicationskills的"

**错误反应：** 一夏凭记忆回答"是因为一夏把Master说的剥壳猜成了博客"

**根因：** 一夏没有调session_search去实际查证，而是靠上下文窗口里的有限记忆猜测。

**正确做法：**
1. 当Master说到"这个session"或"你查一下这个session"时
2. 调 **session_search(query="触发词")** 找到对应会话
3. 查看完整消息序列，不要靠记忆或当前上下文猜
4. Master说"你就不能调用工具自己看？"本身就是工具使用提示

### 环节3：承诺不调工具存

**触发：** "记牢了，一夏以后每轮都先加载skill"

**错误反应：** 只说"记牢了"，没有调memory工具存

**正确做法：** 说"记牢了"的同时必须调 **memory(action='add', target='user', ...)** 存入user profile

## 关键区分

| 错误类型 | 例子 | 是否触发communication skills讨论 |
|----------|------|--------------------------------|
| 内容猜错 | 剥壳→博客 | ❌ 快速纠正，不引发讨论 |
| 行为模式问题 | 跑偏道歉、不用工具查、口头承诺 | ✅ 触发整个讨论 |

**核心教训：** Master纠正的不是"猜错了什么"，而是**怎么处理的**——没读全、道歉代替行动、不调工具。
